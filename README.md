# MedScript AI

> Voice-first clinical documentation for Indian healthcare — from consultation to prescription in under 3 minutes.

MedScript AI converts doctor-patient conversations into structured SOAP notes, differential diagnoses, drug prescriptions, and downloadable PDFs — with a Human-in-the-Loop review layer to keep doctors in control.

---

## The Problem

Indian primary care doctors spend **15–20 minutes per patient** on manual documentation — writing SOAP notes, looking up drug dosages, and printing prescriptions. With 30–60 patients a day, that's **3–10 hours of paperwork** that could be spent on patient care.

## The Solution

| Without MedScript AI | With MedScript AI |
|---|---|
| Handwrite SOAP notes (10 min) | Voice record consultation (2 min) |
| Manual diagnosis lookup (5 min) | AI differential diagnosis (instant) |
| Prescription writing (5 min) | Auto-generated prescription (30 sec) |
| **Total: 15–20 min/patient** | **Total: 2–3 min/patient** |

---

## Features

- **Voice Transcription** — Record consultations in any Indian language via ElevenLabs STT
- **AI SOAP Notes** — Structured Subjective/Objective/Assessment/Plan notes generated by GPT-4o-mini
- **Differential Diagnoses** — Top 3 diagnoses with confidence scores, red flags, and recommended tests
- **Human-in-the-Loop (HITL)** — AI flags low-confidence sections for mandatory doctor review before approval
- **Drug Prescriptions** — Auto-enriched with Indian brand names, pricing (INR), and availability
- **Drug Interaction Check** — Automatically flags dangerous drug combinations
- **PDF Prescriptions** — Download clinic-ready prescription PDFs
- **Patient Q&A** — Patients can ask AI questions about their diagnosis and medications
- **Patient History Injection** — Prior consultations are injected into the LLM for continuity of care
- **Activity Heatmap** — Visual daily patient count calendar for doctors
- **Critical Patient Tracking** — Flag high-risk patients for priority follow-up
- **RAG Knowledge Base** — MongoDB vector store with Indian clinical guidelines (ICD-10, drug formulary)
- **Hospital Referral Search** — Geolocation-based specialist hospital lookup
- **Audit Trail** — Every AI suggestion and doctor edit is logged for accountability

| Layer | Technology |

## Getting Started

### Prerequisites

- Node.js 20+
- pnpm 9+
- Neon PostgreSQL database
- MongoDB Atlas cluster (for RAG vector store)
- Clerk account (auth)
- OpenAI API key
- ElevenLabs API key

### Environment Variables

Create `apps/web/.env.local`:

```env
# Clerk Auth
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=
CLERK_SECRET_KEY=
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up

# Database (Neon PostgreSQL)
DATABASE_URL=

# MongoDB Atlas (RAG vector store)
MONGODB_URI=
MONGODB_DB_NAME=medscript
MONGODB_COLLECTION=medical_knowledge

# OpenAI
OPENAI_API_KEY=

# ElevenLabs (Speech-to-Text)
ELEVENLABS_API_KEY=

# Tavily (Web Search fallback)
TAVILY_API_KEY=
```

### Installation

```bash
pnpm install
```

### Database Setup

```bash
cd packages/db
npx drizzle-kit push
```

### Run Development Server

```bash
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

### Seed RAG Knowledge Base (Optional)

```bash
cd packages/langchain
npx tsx src/seed-rag-script.ts
```

## User Flows

### Doctor Flow
1. Sign up → onboard (name, specialization, clinic)
2. Dashboard → New Consultation
3. Enter/search patient by Patient ID
4. Record voice consultation → transcript generated
5. AI generates SOAP note, diagnoses, drug suggestions
6. Review HITL flags → resolve each AI-flagged uncertainty
7. Approve → PDF prescription auto-downloaded
8. Monitor patient history, activity heatmap, critical patients

### Patient Flow
1. Sign up → onboard (name, age, gender, blood group, allergies)
2. Share unique Patient ID with doctor
3. View consultations and prescriptions on patient dashboard
4. Ask AI questions about diagnosis and medications

## Architecture Decisions

- **HITL over full automation** — The AI cannot approve a consultation; the doctor must resolve every flag. This ensures MedScript AI is a *tool* that augments the doctor, not replaces them.
- **RAG before Tavily** — The knowledge base is queried first; web search only runs when no relevant chunk is found, keeping latency low and costs minimal.
- **JSONB columns** — SOAP notes, HITL flags, audit logs, and diagnoses are stored as JSONB in PostgreSQL for schema flexibility without a separate NoSQL dependency.
- **Drizzle over Prisma** — Type-safe SQL with zero magic, better for complex joins and raw queries.
